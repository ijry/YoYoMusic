# 悠悠乐听提示层硬件化设计

## 背景

五套内置皮肤已经完成机身轮廓、模块舱、控制台、显示窗、内部卡片、信号读数和原生控件残留的多轮打磨。最后仍有一类共享 UI 容易显得像网页组件：空状态、错误提示和皮肤管理说明文案。

本轮只处理这些提示层，让它们成为不同机型上的实体提示硬件，而不是通用文本框换色。

## 范围

本轮覆盖现有提示元素：

- `.empty-state`
- `.error-text`
- `.skin-manager__note`

优先使用 CSS-only 方案，必要时最多补充语义中性的类名。不得修改播放、导入、标签、均衡器、迷你播放器或桌面歌词窗口逻辑。

## 设计目标

- 让空状态像设备的空仓牌、检修窗、提示屏或说明卡。
- 让错误提示像对应机型的告警模块，保留 `role="alert"` 语义。
- 让皮肤管理说明像机器铭牌或参数标签，而不是普通辅助说明文字。
- 五套皮肤必须使用不同结构语言，不只是颜色、阴影或背景渐变差异。
- 保持固定机身壳体，不新增窗口级滚动，不影响底部播放控制区可见性。

## 共享基座

在 `.skin-layout` 作用域内接管提示层的基础外观：

- `.skin-layout .empty-state`
- `.skin-layout .error-text`
- `.skin-layout .skin-manager__note`

共享基座负责建立“提示硬件”的基本属性：可读间距、边框、内嵌表面、字重、较明确的状态色和伪元素插槽。共享层只定义最低共性，不承担具体皮肤形态。

## 皮肤形态

### 经典蓝银分体机

提示层使用细长背光信息窗语言。

- `empty-state` 像卡座旁的蓝色液晶待机窗。
- `error-text` 像红色故障灯条。
- `skin-manager__note` 像银色机身铭牌说明条。

### 暗夜黑胶舱

提示层使用黑胶舞台边缘的霓虹标签语言。

- `empty-state` 像悬浮播报条。
- `error-text` 像红色录音室警示贴。
- `skin-manager__note` 像侧塔服务说明签。

### 透明水晶浮窗

提示层使用玻璃夹层和悬浮胶囊语言。

- `empty-state` 像淡色玻璃提示片。
- `error-text` 像带警戒描边的红色晶体卡。
- `skin-manager__note` 像贴附在浮窗边缘的说明签。

### 金属机架均衡器

提示层使用机架插卡槽和告警模块语言。

- `empty-state` 像空槽位标识。
- `error-text` 像维修告警模块。
- `skin-manager__note` 像机架参数标签。

### 暖木复古唱机

提示层使用纸质说明卡、黄铜标签和木箱内部提示牌语言。

- `empty-state` 像唱片内页留白说明。
- `error-text` 像检修警示小牌。
- `skin-manager__note` 像黄铜铭文卡。

## 实现约束

- 优先只修改 `src/styles/skin-layouts.css`。
- 不修改 Rust/Tauri 播放逻辑。
- 不修改迷你播放器和桌面歌词窗口路由。
- 不引入新依赖。
- 不新增窗口级滚动；仅保留已有内部滚动区域。
- 遵循现有皮肤选择器命名，例如 `.device-shell--classic .empty-state`。
- 如果测试依赖精确选择器匹配，应为皮肤专属规则拆出独立 rule，避免 grouped selector 匹配到错误规则。

## 测试计划

在 `src/styles/skin-layouts.test.ts` 中增加提示层硬件化覆盖：

- 断言共享提示层基座存在。
- 断言五套皮肤分别有 `.empty-state`、`.error-text`、`.skin-manager__note` 的专属硬件化规则。
- 继续保留固定壳体和内部滚动测试。

验证命令：

```powershell
npm test -- src/styles/skin-layouts.test.ts
npm test
npm run build
```

## 验收标准

- 五套皮肤中的空状态、错误提示和说明文案不再呈现为同一套网页提示框。
- 每套皮肤至少体现一种独立的提示硬件形态。
- 错误提示仍可读，并保留现有 `role="alert"`。
- 不出现整窗滚动回归。
- 播放控制区始终可见。
- 全量测试与构建通过。
