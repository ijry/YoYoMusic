# 悠悠乐听原生控件残余收尾设计

**日期:** 2026-07-05
**项目:** 悠悠乐听音乐播放器
**主题:** 收掉五套内置皮肤里最后一批容易暴露“浏览器默认件”来源的细节

## 1. 背景

当前项目已经完成多轮皮肤硬件化工作：

- 五套布局级皮肤已经稳定
- 外壳、模块分仓、控制台、状态窗都已硬件化
- 播放列表、功能区、设置区、皮肤管理区都已做设备化处理
- 歌词、可视化、均衡器的信号读出层也已完成差异化

剩余最容易泄露“这仍然是一个 Web 应用”的细节，已经不再是大模块，而是原生控件残余：

- `checkbox`
- 内部滚动区域的 `scrollbar`
- `disabled` 控件状态

这些细节体量很小，但如果保持浏览器默认感，会削弱整套皮肤“像五台机器”的完成度。

## 2. 目标

本轮目标：

- 收掉最后一批原生控件残余，让五套皮肤在小控件层面也像完整设备。
- 让 `checkbox`、`scrollbar`、`disabled` 态继续服从各皮肤的机型语言。
- 保持当前业务逻辑、布局结构、滚动边界和可访问性不变。

## 3. 非目标

本轮不做：

- 不新增第六套皮肤。
- 不改 JSX 结构。
- 不重构表单组件或抽象通用控件系统。
- 不修改迷你播放器、桌面歌词窗口或 Rust / Tauri 逻辑。
- 不扩展到新的视觉子系统。
- 不恢复整窗滚动。

## 4. 设计结论

本轮采用 **CSS-only 原生控件收尾** 方案。

选择原因：

- 这些问题都集中在已有控件的视觉表层，不需要改结构。
- 通过现有 `.device-shell--*` 包装类就能完成五套机型差异。
- 范围小、风险低，适合作为最终验收前的最后一轮收口。

## 5. 设计范围

本轮仅处理：

- `checkbox`
  - 目前主要出现在 `均衡器启用` 开关
- `scrollbar`
  - 播放列表区
  - 歌词区
  - 功能面板内部可滚动区
- `disabled`
  - 运输控制按钮
  - 小型操作按钮
  - 预设按钮和其他可禁用按钮

## 6. 每套皮肤的控件语义

### 6.1 经典蓝银分体机

目标语义：冷光仪表件与精密导槽。

强化点：

- `checkbox` 像面板里的小型拨定块。
- `scrollbar` 像嵌入式导轨与细刻度滑槽。
- `disabled` 态像待机但仍有硬件存在感的冷金属按钮。

### 6.2 暗夜黑胶舱

目标语义：舞台胶囊件与暗色光带。

强化点：

- `checkbox` 更像圆角拨片或 stage toggle。
- `scrollbar` 更像发光窄轨，而不是默认滚动条。
- `disabled` 态保持暗场设备感，不变成灰白网页按钮。

### 6.3 透明水晶浮窗

目标语义：玻璃拨片与半透明滑轨。

强化点：

- `checkbox` 像悬浮玻璃小片。
- `scrollbar` 像玻璃边缘中的透明控制轨。
- `disabled` 态保留玻璃存在感，只降低活性，不直接“发灰”。

### 6.4 金属机架均衡器

目标语义：机架校准件与工业导轨。

强化点：

- `checkbox` 像设备面板上的校准开关。
- `scrollbar` 更像机柜里的滑轨槽与信号标尺。
- `disabled` 态像断电但仍保有工业轮廓的机架按钮。

### 6.5 暖木复古唱机

目标语义：黄铜细件与木箱槽口。

强化点：

- `checkbox` 像小型黄铜拨片。
- `scrollbar` 像木箱里的细槽或暖光导轨。
- `disabled` 态更像收起或待机的实体件，而不是浏览器灰态。

## 7. 实现策略

本轮继续坚持 CSS-only：

- 优先修改 `src/styles/skin-layouts.css`
- 必要时只做极少量 `src/styles/app.css` 基底收口
- 补充 `src/styles/skin-layouts.test.ts` 的样式合同

实现方式：

- 用已有皮肤变量和 `.device-shell--*` 包装类覆盖原生控件表层
- 为内部滚动区补统一但分皮肤的滚动条轨道 / 滑块语义
- 为 `disabled` 态补“设备待机态”而不是“网页不可用态”

## 8. 约束

继续沿用主规格中的全局约束：

- `body` 不允许恢复滚动。
- 最外层应用容器不允许因为内容变长而撑高。
- 允许滚动的区域仅限：播放列表区、歌词区、功能面板区、少数设置项较长的子面板。
- 播放 / 暂停、上一首、下一首、音量、进度始终可见且可访问。
- `prefers-reduced-motion` 下关闭扫描光、脉冲灯、频闪类装饰效果。

补充约束：

- 不能因为重新画 `checkbox` 而降低键盘和屏幕阅读器可用性。
- 滚动条需要可见，但不能过重到影响内容阅读。
- `disabled` 态必须仍然清晰表达不可操作，不得只顾装饰忽略可理解性。

## 9. 测试策略

自动化测试继续验证：

- 新增 CSS 合同断言，锁定原生控件收尾钩子。
- 保持当前布局 / 样式 / 组件测试不回归。
- `npm test` 与 `npm run build` 继续通过。

人工验收重点：

- 五套皮肤的 `checkbox`、滚动条、禁用按钮不再有明显浏览器默认件观感。
- 各类内部滚动区仍然清晰、可用、不抢内容视觉。
- `disabled` 态依然一眼能看出不可点击。

## 10. 影响文件

预计涉及：

- `src/styles/skin-layouts.css`
- `src/styles/skin-layouts.test.ts`
- 可能少量涉及 `src/styles/app.css`

## 11. 验收标准

完成后应满足：

- 最后一批原生控件残余被收掉。
- 五套皮肤在小控件层面也维持完整设备语义。
- 不改变当前功能和结构。
- 不引入整窗滚动回归。
- `npm test` 和 `npm run build` 通过。
