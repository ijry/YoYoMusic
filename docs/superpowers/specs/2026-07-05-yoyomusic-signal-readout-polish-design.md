# 悠悠乐听信号读出层硬件化收尾设计

**日期:** 2026-07-05
**项目:** 悠悠乐听音乐播放器
**主题:** 继续强化五套内置皮肤的“信号读出层”，让歌词、可视化和均衡器不再像同一套网页面板

## 1. 背景

当前五套内置皮肤已经完成：

- 外壳轮廓与机身硬件层
- 模块分仓与标题舱
- 控制台与状态窗
- 播放列表、功能区、设置区、皮肤管理区的内部卡片层

剩余最容易暴露“同一个 Web 应用换皮肤”感的区域，集中在信号读出层：

- 歌词区的逐行读出
- 可视化面板的模式按钮与频谱柱条容器
- 均衡器的启用开关、预设按钮、频段卡片和滑杆

这些区域虽然已有基础皮肤化，但仍共享较多同构的表面语言，缺少各机型独立的读出设备语义。

## 2. 目标

本轮目标：

- 强化五套皮肤在信号读出层的差异，让它们看起来像五种不同的读出设备。
- 优先消除歌词行、频谱读出和 EQ 调节区的“统一网页卡片”感。
- 保持当前布局结构、业务逻辑和窗口行为不变。
- 继续让默认皮肤 `经典蓝银分体机` 保持最强的桌面旗舰播放器气质。

## 3. 非目标

本轮不做：

- 不新增第六套皮肤。
- 不改 JSX 信息架构，不新增复杂包装结构。
- 不重构播放、均衡器、歌词或可视化逻辑。
- 不修改迷你播放器或桌面歌词窗口。
- 不恢复整窗滚动。
- 不引入新依赖、图片贴图或外部字体。

## 4. 设计结论

本轮采用 **纯 CSS 信号读出硬件化** 方案。

选择原因：

- 能直接沿用现有 `.lyric-line`、`.visualization-preview--panel`、`.equalizer-panel__toggle`、`.eq-band-card` 等稳定钩子。
- 改动范围小，风险低，不需要再触碰 React 组件结构。
- 对用户的主观观感收益足够高，能把最后几块“共享表面”压下去。

## 5. 设计范围

本轮仅处理以下表面：

- `歌词`
  - `.lyrics-panel__viewport`
  - `.lyric-line`
  - `.lyric-line__stamp`
- `可视化`
  - `.visualization-panel__meter`

  - `.equalizer-panel__toggle`

  - `input[type="range"]`

## 6. 每套皮肤的读出语义

### 6.1 经典蓝银分体机

目标语义：冷光 LCD 与精密刻度读出。

强化点：

- 歌词行更像排布在蓝背光信息屏上的读数槽。
- 可视化柱条更像嵌在显示桥中的冷光频谱。
- EQ 滑杆更像面板内的细刻度推杆，带精密仪表感。

### 6.2 暗夜黑胶舱

目标语义：舞台字幕屏与圆角胶囊读数。

强化点：

- 歌词行改为更连续的舞台提词条带，而不是普通卡片堆叠。
- 可视化柱条更强调节奏感与夜场光带读数。
- EQ 调节区保持胶囊与弧形语义，像表演模式控制条。

### 6.3 透明水晶浮窗

目标语义：悬浮玻璃示波层。

强化点：

- 歌词行像漂浮在玻璃中的读数片。
- 可视化表盘与柱条像悬浮 HUD 频谱层。
- EQ 滑杆与预设区像透明玻璃调校片，不是实体按钮网格。

### 6.4 金属机架均衡器

目标语义：LED / VU 机柜读出与机架校准条。

强化点：

- 歌词行更像机柜中的窄条读出槽。
- 可视化更像设备面板内的 LED 柱表。
- EQ 区域最强调专业感，滑杆和读数像音频机架校准面板。

### 6.5 暖木复古唱机

目标语义：背光铭牌与木箱拨杆面板。

强化点：

- 歌词行像暖色铭牌或节目单条。
- 可视化面板像嵌在木箱中的暖光观察窗。
- EQ 控件像木箱里的调音拨片与黄铜刻度，不出现现代电子感过强的表面。

## 7. 实现策略

本轮实现坚持 CSS-only：

- 只修改 `src/styles/skin-layouts.css`
- 只补充 `src/styles/skin-layouts.test.ts` 中的样式合同
- 不改 `LyricsPanel.tsx`
- 不改 `VisualizationPanel.tsx`
- 不改 `EqualizerPanel.tsx`

实现方法：

- 通过伪元素、渐变、边框层、内阴影和局部圆角差异强化读出件语义。
- 通过每套 `.device-shell--*` 包装类覆盖共享表面。
- 对滑杆、柱条和活跃歌词行增加材质语义，不依赖持续动画。

## 8. 约束

继续沿用主规格中的全局约束：

- `body` 不允许恢复滚动。
- 最外层应用容器不允许因为内容变长而撑高。
- 允许滚动的区域仅限：播放列表区、歌词区、功能面板区、少数设置项较长的子面板。
- 播放 / 暂停、上一首、下一首、音量、进度始终可见且可访问。
- `prefers-reduced-motion` 下关闭扫描光、脉冲灯、频闪类装饰效果。

补充约束：

- 不能因为强调读出层而降低歌词可读性。
- 滑杆和按钮的命中区不能缩小。
- 小窗口下必须优先保留信息可读性，必要时隐藏大装饰伪元素。

## 9. 测试策略

自动化测试继续验证：

- 新增 CSS 合同断言，锁定五套皮肤的信号读出差异钩子。
- 现有布局与皮肤管理相关测试不回归。
- `npm test` 与 `npm run build` 继续通过。

人工验收重点：

- 五套皮肤的歌词、可视化和 EQ 区一眼看出来自不同设备。
- 经典蓝银更像旗舰播放器的读出桥。
- 金属机架的 EQ 专业感最强。
- 水晶与木质两套皮肤的读出层仍然清晰，不会因为装饰过重影响阅读。

- 不改变现有功能和交互逻辑。
- 不引入整窗滚动回归。
- `npm test` 和 `npm run build` 通过。
