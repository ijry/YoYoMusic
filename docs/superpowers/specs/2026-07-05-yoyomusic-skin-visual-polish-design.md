# 悠悠乐听五套内置皮肤硬件化视觉强化设计

**日期:** 2026-07-05
**项目:** 悠悠乐听音乐播放器
**主题:** 在既有五套布局级皮肤基础上，进一步强化“老式播放器外壳”质感，让皮肤不仅布局不同，而且像五台不同机型的播放器

## 1. 背景

当前项目已经完成五套布局级皮肤：

- `classic-blue-silver`
- `dark-vinyl`
- `transparent-crystal`
- `metal-rack`
- `warm-wood`

这些皮肤已经具备真实布局差异，但视觉层仍偏现代 Web 面板组合。用户进一步要求参考千千静听时代的内置皮肤体验，使每套皮肤拥有更强的播放器外壳感、设备感和机身分仓感，不能看起来只是“换了布局的网页”。

本轮工作是对既有布局皮肤进行视觉深化，而不是重新设计产品信息架构。

## 2. 目标

本轮目标：

- 强化五套皮肤的机身轮廓、面板边界、控制件造型和显示窗语义。
- 让五套皮肤像五台不同风格、不同年代、不同定位的播放器设备。
- 保持当前布局系统和共享业务组件复用，不推翻既有布局成果。
- 继续保证主窗口是固定工作台，不恢复整窗滚动。
- 让默认皮肤 `经典蓝银分体机` 更接近“千千静听时代桌面旗舰播放器”的感觉。

## 3. 非目标

本轮不做：

- 不增加第六套皮肤。
- 不重构 Tauri / Rust 播放逻辑。
- 不改变迷你播放器或桌面歌词窗口。
- 不引入图片贴图、外部字体依赖或新的 UI 库。
- 不做像素级仿制某一款历史软件。
- 不把皮肤系统扩展为第三方布局 DSL。

## 4. 设计结论

本轮采用 **A. 分体机硬件化** 方向，作为五套皮肤统一的视觉强化原则。

选择这一方向的原因：

- 最贴合“像老式硬件 / 播放器外壳”的用户要求。
- 能覆盖五套现有布局，不需要重写布局组件。
- 能让不同皮肤共享“设备化”语言，同时保持各自机型差异。
- 风险可控，主要集中在 CSS 和少量包装结构，不触及业务逻辑。

## 5. 整体视觉语言

### 5.1 皮肤不再被视为页面主题

每套皮肤应被设计成一台完整设备，而不是一个套了配色的应用页面。

要求：

- 整个主窗口要有机身轮廓。
- 功能模块要像镶嵌在机身里的仓段。
- 控制条要像控制台，而不是普通网页工具栏。

### 5.2 外壳层

五套皮肤都必须具备独立外壳语言：

- 明确的外框、压边、倒角或舱体结构。
- 通过伪元素、边框层、内阴影和渐变营造机身厚度。
- 不再只有单一圆角卡片叠放关系。

### 5.3 面板层

播放列表、当前播放、功能区、可视化区应被包装成设备模块，而不是无语义分栏。

要求：

- 模块之间具备拼装感，例如隔条、框槽、仓门、浮仓或铭牌槽。
- 模块标题使用“设备标签”语气，例如状态窗、曲目仓、控制台、频谱桥。
- 模块装饰集中在边界，不侵入内容阅读区。

### 5.4 控制件层

播放控制、音量和进度条要强化为设备控制件视觉语言。

要求：

- 播放控制按钮有按键帽、压下高光或金属边圈感。
- 音量优先强化为旋钮语义。
- 进度条优先强化为导轨、刻度槽或推杆语义。
- 各皮肤控制件造型可不同，但主控制区必须保持一眼可识别。

### 5.5 显示层

当前歌曲信息、播放状态、时间和模式文字要更像嵌入式显示窗。

五套皮肤的显示层建议：

- `classic-blue-silver`: 冷光 LCD 状态窗
- `dark-vinyl`: 暗色舞台字幕屏
- `transparent-crystal`: 透明 HUD 信息层
- `metal-rack`: LED / VU 仪表式显示窗
- `warm-wood`: 暖黄背光铭牌窗

### 5.6 装饰层

允许出现的设备细节包括：

- 拉丝金属纹
- 玻璃高光边
- 木纹包边
- 螺丝位
- 刻度线
- 状态灯
- 通风孔
- 编号铭牌

约束：

- 装饰只服务于机壳识别。
- 不得遮挡文本、控件和频谱内容。
- 不通过大量动画制造“设备感”。

## 6. 五套皮肤的具体强化方向

### 6.1 经典蓝银分体机

定义为默认旗舰机型。

强化点：

- 顶部主控舱采用蓝背光状态屏、银色压边和机型铭牌槽。
- 左侧播放列表改造为抽屉式曲目仓，带导轨和凹槽感。
- 中央当前播放区强化为主显示舱，封面和信息像主机显示面板。
- 右侧功能区强化为扩展功能仓，保留标签但增加设备面板语义。
- 底部控制台采用最强机械感方案：大号播放键、金属环音量旋钮、导轨式进度和小状态灯。

### 6.2 暗夜黑胶舱

定义为沉浸式播放机型。

强化点：

- 中央唱盘 / 封面区采用嵌入式圆形舞台，外圈带压环和刻纹。
- 当前歌曲信息使用暗色数码字幕窗。
- 右侧歌词和列表合并为竖向设备塔，减少普通列表页感。
- 控制区围绕唱盘组织，优先使用弧形或环抱式控制节奏。
- 可视化强调环形呼吸感和底部舞台频谱，不依赖强动画。

### 6.3 透明水晶浮窗

定义为概念型透明设备。

强化点：

- 外壳使用厚边透明框、分层板件和内发光边，而不是单纯玻璃卡片。
- 播放列表塑造成透明抽拉资料匣。
- 功能区塑造成悬浮胶囊模块，形成明显层差。
- 当前播放信息像浮在玻璃中层的 HUD，不直接贴平面底板。
- 底部控制台像一块悬浮透明底座，继续保持固定可见。

### 6.4 金属机架均衡器

定义为专业设备机型。

强化点：

- 整机更方、更硬、更工业化，接近 19 英寸音频机架语义。
- 上层主视觉是双 VU 表或 LED 频谱桥。
- 中层是播放状态显示窗。
- 下层是控制区与信息机柜屏，列表像设备状态面板而非侧栏。
- 使用机架隔条、螺丝位、编号铭牌和散热孔建立分仓感。

### 6.5 暖木复古唱机

定义为家居陈设型机型。

强化点：

- 外壳以木质箱体、黄铜面板和暖光信息窗为主。
- 左侧封面区像唱片套插槽或陈列窗。
- 右侧列表像节目单 / 专辑内页，但保持数字交互清晰。
- 控制件采用象牙白按键、黄铜旋钮和细拨杆语义。
- 整体弱化冷光和霓虹效果，强调暖色、材质和陈设感。

## 7. 结构与滚动约束

### 7.1 固定工作台

所有皮肤继续遵守主窗口固定工作台原则：

- `body` 不允许恢复滚动。
- 最外层应用容器不允许因为内容变长而撑高。
- 用户感受到的是一台桌面播放器，而不是一个页面。

### 7.2 内部滚动边界

允许滚动的区域仅限：

- 播放列表区
- 歌词区
- 功能面板区
- 少数设置项较长的子面板

默认不滚动的区域：

- 顶部主控舱
- 当前播放主显示舱
- 控制台
- 主要可视化框体

### 7.3 响应式退化

当窗口尺寸收窄时，允许仓位重排，但必须保持机型感。

规则：

- 中宽度下可将部分并排仓位变为上下堆叠。
- 小宽度下可退化为单列，但仍保留标题舱、主显示舱、控制台三段式骨架。
- 优先压缩装饰和次要说明，不优先压缩核心控制区。

## 8. 交互与可访问性

要求：

- 播放 / 暂停、上一首、下一首、音量、进度始终可见且可访问。
- 皮肤切换后不改变基础操作习惯。
- 焦点样式继续清晰可见。
- `prefers-reduced-motion` 下关闭扫描光、脉冲灯、频闪类装饰效果。
- 不能只靠颜色区分当前皮肤或当前状态，仍保留文本状态。

## 9. 实现策略

本轮实现以样式和轻量结构包装为主，不重写业务组件。

建议实现方式：

- 继续使用现有五套 layout 组件。
- 在各 layout 内增加少量包装层、标题语义层和装饰钩子类名。
- 主要修改集中在 `src/styles/skin-layouts.css`。
- 必要时微调 `src/features/skin/layoutShared.tsx` 和 `src/features/skin/layouts.tsx`，以支持更清晰的设备模块结构。
- 不改变 `App.tsx` 的皮肤选择主流程。

## 10. 测试策略

自动化测试继续验证：

- 五套皮肤关键 landmark 仍存在。
- 播放控制、进度、音量在所有皮肤下可访问。
- 主窗口不恢复整窗滚动。
- 皮肤切换行为不回归。

人工验收重点：

- 五套皮肤一眼看出是不同机型，而非同一布局换色。
- `经典蓝银分体机` 最具千千静听时代桌面旗舰感。
- 所有装饰均未破坏文本可读性和控件命中区。
- 任何长列表或长面板都只在内部滚动。

## 11. 影响文件

预计涉及：

- `src/styles/skin-layouts.css`
- `src/features/skin/layouts.tsx`
- `src/features/skin/layoutShared.tsx`
- `src/styles/skin-layouts.test.ts`
- `src/styles/app-layout.test.ts`
- 可能少量涉及 `src/features/skin/SkinManager.tsx`，用于同步皮肤文案描述

## 12. 验收标准

完成后应满足：

- 五套皮肤都具备明确的设备外壳感和面板分仓感。
- 五套皮肤不只是颜色不同，而是机身轮廓、控制台语义和显示窗语义都不同。
- 默认皮肤 `经典蓝银分体机` 呈现为最完整的老式桌面旗舰播放器形象。
- 在所有皮肤下，播放列表、当前播放、控制台、音量、功能面板和可视化仍然清晰可用。
- 主窗口继续保持固定，不出现整窗滚动回归。
- `npm test` 和 `npm run build` 在后续实现完成后通过。
